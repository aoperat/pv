# 배포 완료 체크리스트 ✅

## 1. 배포 상태 확인

✅ **빌드 완료**: `npm run build` 성공
✅ **gh-pages 푸시 완료**: "Published" 메시지 확인됨

## 2. GitHub Pages 설정 (필수!)

**다음 링크에서 설정하세요:**
https://github.com/aoperat/pv/settings/pages

**설정 내용:**
- Source: Deploy from a branch
- Branch: `gh-pages` 선택
- Folder: `/ (root)` 선택
- Save 클릭

## 3. 배포 주소

설정 완료 후 접속 가능:
**https://aoperat.github.io/pv/**

## 4. 배포 상태 확인 방법

### 방법 1: GitHub에서 확인
- 저장소 페이지 오른쪽 사이드바 > "Environments" 섹션
- "github-pages" 환경이 "Active" 상태인지 확인

### 방법 2: Settings > Pages에서 확인
- "Your site is live at https://aoperat.github.io/pv/" 메시지 확인

### 방법 3: 브라우저에서 직접 접속
- https://aoperat.github.io/pv/ 접속
- 페이지가 로드되면 성공!

## 5. 문제 해결

### 페이지가 안 보일 때:
1. Settings > Pages에서 Source가 `gh-pages`로 설정되었는지 확인
2. 몇 분 더 기다리기 (배포에 시간이 걸릴 수 있음)
3. 브라우저 캐시 삭제 후 다시 시도 (Ctrl + Shift + R)

### 404 에러가 나올 때:
- URL이 정확한지 확인: `https://aoperat.github.io/pv/`
- 저장소 이름이 대소문자 정확히 일치하는지 확인 (소문자 `pv`)

### 빌드 에러가 날 때:
```bash
# 로컬에서 먼저 테스트
npm run build
npm run preview
```

## 6. 업데이트 배포 방법

코드를 수정한 후 다시 배포:
```bash
git add .
git commit -m "Update content"
git push origin main
npm run deploy
```

## 7. 환경변수 확인

⚠️ **중요**: 배포된 사이트에서 Supabase가 작동하려면:
- `.env.local`의 값이 빌드 시 포함되었는지 확인
- 또는 GitHub Actions 방법을 사용할 경우 GitHub Secrets 설정 필요

현재는 수동 배포 방법이므로 `.env.local`의 값이 빌드에 포함되었습니다.
